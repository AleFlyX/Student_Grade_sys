
# 学生选课系统用例规约

## 1. 学生登录

### 简要说明
本用例允许学生通过学号和密码登录系统，验证身份后进入学生个人主页。

### 事件流
#### 2.1 基本事件流
1. 学生进入系统登录页面
2. 系统显示登录表单（学号、密码输入框）
3. 学生输入学号和密码
4. 系统验证学号和密码的正确性
5. 验证通过，系统跳转到学生个人主页

#### 2.2 备选事件流
1. 学号或密码为空：系统提示"请输入学号和密码"
2. 学号不存在：系统提示"学号不存在"
3. 密码错误：系统提示"密码错误，请重试"

### 特殊需求
无

### 前置条件
学生已注册账户，系统正常运行

### 后置条件
登录成功后，学生处于已登录状态，可访问学生相关功能


## 2. 学生注册

### 简要说明
本用例允许新生或未注册学生创建账户，完成个人信息录入。

### 事件流
#### 2.1 基本事件流
1. 用户进入注册页面
2. 系统显示注册表单（学号、姓名、密码、确认密码、联系方式等）
3. 用户填写注册信息
4. 系统验证信息完整性和格式正确性
5. 验证通过，系统创建学生账户
6. 系统提示注册成功

#### 2.2 备选事件流
1. 学号已存在：系统提示"该学号已注册"
2. 密码不一致：系统提示"两次输入的密码不一致"
3. 信息不完整：系统提示"请填写完整信息"

### 特殊需求
密码需符合复杂度要求（至少8位，包含字母和数字）

### 前置条件
学生未注册账户，系统正常运行

### 后置条件
注册成功后，学生账户创建完成，可进行登录


## 3. 查看/修改个人信息

### 简要说明
本用例允许学生查看和修改个人基本信息，如联系方式、地址等。

### 事件流
#### 2.1 基本事件流
1. 学生进入个人信息页面
2. 系统显示学生当前个人信息
3. 学生点击"修改"按钮
4. 系统显示可编辑的信息表单
5. 学生修改信息并提交
6. 系统验证并保存修改

#### 2.2 备选事件流
1. 修改信息格式错误：系统提示"信息格式不正确"
2. 修改信息失败：系统提示"修改失败，请重试"

### 特殊需求
学号不可修改

### 前置条件
学生已登录系统

### 后置条件
个人信息更新成功，系统保存新信息


## 4. 查看可选课程列表

### 简要说明
本用例允许学生查看当前学期开设的所有可选课程列表。

### 事件流
#### 2.1 基本事件流
1. 学生进入选课页面
2. 系统查询当前学期的课程信息
3. 系统显示课程列表（包含课程编号、名称、学分、授课教师、上课时间等）
4. 学生可按条件筛选（如课程类别、学分等）

#### 2.2 备选事件流
1. 当前学期无可选课程：系统提示"暂无可选课程"

### 特殊需求
支持课程搜索和筛选功能

### 前置条件
学生已登录系统，选课时间未结束

### 后置条件
学生可查看可选课程列表


## 5. 选课

### 简要说明
本用例允许学生选择当前学期提供的课程，在选课期间可添加课程到个人课表。

### 事件流
#### 2.1 基本事件流
1. 学生在课程列表中选择一门或多门课程
2. 系统检查课程是否可选（未选满、时间不冲突）
3. 学生确认选课
4. 系统将课程添加到学生课表
5. 系统提示选课成功

#### 2.2 备选事件流
1. 课程已选满：系统提示"该课程已选满"
2. 时间冲突：系统提示"所选课程与已选课程时间冲突"
3. 已达到学分上限：系统提示"已达到学期学分上限"

### 特殊需求
同一门课程不可重复选择

### 前置条件
学生已登录系统，选课时间未结束

### 后置条件
选课成功后，学生课表更新，课程人数更新
