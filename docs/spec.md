**大学生课程成绩查询系统**，需要体现大学特色：学分、绩点（GPA）、选课关联、补考/重修、教务审核、成绩等级等。

# 大学生课程成绩查询系统 - 需求规格说明书 (V1.0)

## 1. 引言

### 1.1 项目背景
为满足本校大学生、任课教师及教务管理人员随时随地在线查询、录入和管理课程成绩的需求，替代传统纸质成绩单，支持学分制下的绩点计算与学业预警，拟开发本系统。

### 1.2 项目目标
- 安全的角色登录与认证（学生、教师、教务管理员）。
- 学生可查询本人所有修读课程的成绩、学分、绩点及 GPA。
- 教师可录入、修改所授课程的成绩（需教务审核生效）。
- 支持补考、重修成绩的单独标记与展示。
- 提供成绩统计分析（专业排名、分数分布等）。
- 保证成绩数据的准确性、隐私性及查询效率。

### 1.3 术语定义
- **学生**：在校本科生/研究生，可查询自己的成绩。
- **任课教师**：承担某门课程教学任务的教师，可录入/修改该课程的成绩。
- **教务管理员**：教务处人员，负责审核成绩、发布成绩、管理课程/学生/教师数据。
- **课程**：含课程编号、名称、学分、课程性质（必修/选修/公选）、开课学期。
- **选课记录**：学生与课程的关联，是成绩的载体。
- **成绩**：包含总评成绩（分数或等级）、平时/期末/实验等分项成绩、补考/重修标记。
- **绩点**：根据成绩分数或等级换算的绩点（通常 0~4.0 或 0~5.0）。
- **GPA**：平均学分绩点 = ∑(课程绩点×学分) / ∑学分。

## 2. 用户角色

| 角色 | 描述 | 核心权限 |
|------|------|----------|
| 学生 | 在校学生 | 查询本人成绩、学分、GPA、成绩单打印（PDF） |
| 任课教师 | 课程授课教师 | 录入/修改所授课程的成绩（提交后待教务审核）<br>查询所授课程的学生成绩列表及统计 |
| 教务管理员 | 教务处工作人员 | 课程管理、选课数据管理、成绩审核与发布、成绩修改审批、系统配置 |

## 3. 功能需求

### 3.1 登录与身份认证
- **功能描述**：用户通过学号/工号和密码登录系统。
- **详细要求**：
  - 学生用学号，教师用工号，教务管理员用专用账号。
  - 密码加密存储，支持找回密码（通过校内邮箱/手机）。
  - 登录失败5次锁定15分钟。
  - 登录后跳转至对应角色的成绩查询主页。

### 3.2 成绩查询（学生端核心）
- **功能描述**：学生按条件查询已发布（或全部）的课程成绩，并自动计算 GPA。
- **查询筛选条件**：
  - 学年（如2024-2025）
  - 学期（秋季、春季、夏季）
  - 课程性质（必修/选修/公选）
  - 课程名称模糊搜索
  - 成绩状态（正常/补考/重修）
- **查询结果展示（表格）**：
  - 课程编号、课程名称、学分、课程性质、总评成绩（百分制/等级）、绩点、成绩状态（首次/补考/重修）、录入教师、考试时间
  - 支持按学年/学期分组折叠显示
  - 底部汇总：总修读学分、获得学分、加权平均分、GPA
- **成绩详情**：
  - 点击课程可查看分项成绩（平时成绩、期中成绩、期末成绩、实验成绩等）及教师评语。
- **补考/重修逻辑**：
  - 若某学生某课程有多次修读记录（正常、补考、重修），**系统应显示所有记录**，并明确标注“首次修读”、“补考1”、“重修1”等。
  - **GPA计算按最高一次成绩**或学校规定策略（需配置）。

### 3.3 GPA与学业分析（学生端）
- **自动计算GPA**：根据学校绩点换算规则（如95-100→4.0，90-94→3.7等），按学分加权计算。
- **可视化图表**：
  - 各学期 GPA 走势折线图
  - 各课程性质（必修/选修）学分占比饼图
  - 与专业平均 GPA 对比（若提供专业数据）
- **学业预警**：当累计不及格学分 ≥ 某阈值（如15学分），系统弹出提示并建议联系学业导师。

### 3.4 成绩录入与管理（教师端 + 教务端）

#### 3.4.1 成绩录入（任课教师）
- **功能描述**：教师选择某学期所授课程，按选课学生名单录入成绩。
- **详细要求**：
  - 支持百分制分数或等级制（优、良、中、及格、不及格），支持小数（如89.5）。
  - 支持平时/期末/实验分项成绩设置及比例自动计算总评。
  - 暂存功能：教师可多次录入，确认无误后“提交审核”。
  - 提交后成绩变为“审核中”，不可再修改（除非教务退回）。

#### 3.4.2 成绩审核（教务管理员）
- **功能描述**：教务对教师提交的成绩进行审核。
- **操作**：
  - 通过：成绩正式生效，学生端可见。
  - 退回：附理由，教师修改后重新提交。
- **批量审核**：支持按课程或按院系批量操作。

#### 3.4.3 成绩修改申请（特殊流程）
- 学生若发现成绩错误，可在系统内提起“成绩复核申请”，经任课教师、教务两级审批后，教务管理员直接修改成绩并记录日志。

### 3.5 成绩统计与分析（教师/教务端）
- **教师端**：
  - 所授课程的成绩分布直方图（分数段人数）、平均分、最高/最低分、标准差。
  - 导出成绩单（Excel/PDF）用于归档或报送。
- **教务端**：
  - 全校/院系/专业成绩分析：挂科率、优秀率、平均GPA排名等。
  - 支持按年级、专业生成报表。

### 3.6 成绩单自助打印（学生端）
- **功能描述**：生成带学校电子签章的正式成绩单（PDF、防伪水印）。
- **内容包含**：学生信息、所有课程成绩（按学期排序）、学分统计、GPA。
- **支持防伪二维码**：扫描可联网验证真伪。

### 3.7 数据导入/导出（教务端）
- **批量导入**：Excel 导入课程、学生选课名单、历史成绩（对接教务系统）。
- **批量导出**：按专业、年级导出所有学生成绩。

## 4. 非功能需求

### 4.1 性能要求
- 登录响应 ≤ 2 秒。
- 成绩查询（学生4年全部课程）响应 ≤ 3 秒。
- 支持并发在线 ≥ 2000 人（按本科在校生1万计，高峰期约20%在线）。

### 4.2 安全性
- 密码 bcrypt 加密，HTTPS 传输。
- 学生只能看自己的成绩；教师只能看所授课程的成绩；教务可看所有。
- 成绩修改（包括教师提交、教务审核、复核修改）必须记录完整日志（操作人、时间、原值、新值、原因）。

### 4.3 可用性与易用性
- 响应式设计，适配手机端查询成绩。
- 提供 GPA 计算规则说明悬浮帮助。
- 对于不及格/补考成绩，用红色标记提醒。

### 4.4 可维护性与扩展性
- 前后端分离，绩点计算规则应可配置（支持多种换算标准）。
- 预留对接学校统一身份认证（CAS/OAuth）接口。

## 5. 数据需求

### 5.1 核心数据实体
- **用户表**：user_id，角色（学生/教师/教务），登录账号（学号/工号），密码，姓名，所属院系，邮箱/手机。
- **学生信息表**：关联 user_id，入学年份，专业，班级（行政班），培养方案版本。
- **教师信息表**：关联 user_id，职称，所属院系。
- **课程信息表**：课程编号，课程名称，学分，课程性质（必修/选修），开课院系，总学时。
- **开课记录表**（教学班）：学期，课程编号，任课教师ID，上课时间地点，选课容量。
- **选课记录表**：学生ID，开课记录ID，是否放弃考试等状态。
- **成绩表**：选课记录ID，总评成绩（数值/等级），平时成绩，期末成绩，实验成绩，成绩状态（首次/补考/重修），补考/重修成绩，录入教师，审核状态（草稿/待审核/已通过/已退回），生效时间。
- **绩点换算规则表**：分数下限，分数上限，等级（A/B/C），绩点值，允许自定义。

### 5.2 数据约束
- 同一学生、同一课程（同一开课记录）只能有一条正式成绩记录（补考/重修视为不同修读次数，可有多条记录，但需关联不同的学期或标记）。
- 成绩分数范围 0-100，等级在预设枚举值内。
- GPA 按学校规则实时计算，不持久化存储。

## 6. 约束与假设

- 假设学校已有统一身份认证及教务系统中的选课数据，本系统通过接口或数据同步获得选课关系。
- 成绩录入的截止日期、补考成绩录入由教务在后台配置。
- 本系统不涉及学生选课操作，仅用于成绩查询与管理。
- 家长不单独设账号，大学生已成年，家长需求由学生自行转达。

## 7. 用户界面原型描述（简略）

- **登录页**：学号/工号，密码，角色下拉（学生/教师/教务管理员）。
- **学生端首页**：顶部显示姓名、学号、当前学期 GPA；中间为查询筛选栏；下方为课程成绩列表（可展开详情）；右侧显示学分统计卡片。
- **教师端成绩录入页**：选择课程学期 → 显示学生名单表格（学生、学号、平时、期末、实验、总评）→ 批量或逐个录入 → 暂存/提交审核。
- **教务管理后台**：待审核课程列表，可筛选院系/课程；成绩复核申请审批页面；成绩导入/导出界面。

## 8. 未来可扩展功能（V2.0）

- 移动App（可用小程序快速查询成绩及GPA）
- 成绩排名（专业排名、年级排名，学生端自愿开启）
- 与学工系统对接，自动推送给学业预警学生辅导员
- 成绩证明在线验证平台（类似学信网）
